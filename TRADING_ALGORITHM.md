# 🚀 암호화폐 거래 알고리즘 - 정밀 수익 전략

## 📌 핵심 전략: "다중 신호 기반 하이브리드 거래법"

정밀하고 거래량을 활용한 최적의 알고리즘입니다.

---

## 🎯 거래 신호 구성 (4가지 주요 요소)

### 1️⃣ **트렌드 확인 (Trend Confirmation)**
```
조건: 3단계 이동평균 정렬
- 단기 EMA(12) > 중기 EMA(26) > 장기 SMA(50) = 강한 상승
- 역순 = 강한 하락

사용 이유: 
✓ 메인 트렌드 방향 확인
✓ 추세와 반대로 거래하는 실수 방지
```

### 2️⃣ **모멘텀 판단 (Momentum - MACD + RSI)**
```
MACD (12, 26, 9):
- MACD > Signal Line & 0 위: 상승 모멘텀
- MACD < Signal Line & 0 아래: 하락 모멘텀
- 히스토그램 크기: 모멘텀 강도

RSI (14):
- 30 이하: 과도한 하락 (매수 기회)
- 70 이상: 과도한 상승 (매도 기회)
- 50 근처: 중립
```

### 3️⃣ **거래량 검증 (Volume Confirmation)** ⭐ 가장 중요
```
거래량 기반 신뢰도 검증:
- Volume_MA20 = 지난 20일 평균 거래량
- 현재 거래량 > Volume_MA20 × 1.5 = 신뢰도 높음

매매 신호 신뢰도:
📊 거래량 < 평균: 신호 신뢰도 ⭐ (낮음)
📊 거래량 = 평균: 신호 신뢰도 ⭐⭐⭐ (중간)
📊 거래량 > 1.5×평균: 신호 신뢰도 ⭐⭐⭐⭐⭐ (매우 높음)
```

### 4️⃣ **가격 액션 (Price Action)**
```
캔들 패턴:
- 상승: 실체 > 상단 심지 (강한 매수)
- 하락: 실체 < 하단 심지 (강한 매도)
- 망치: 저점 터치 후 반등 (바닥 신호)
- 별: 방향 전환 신호
```

## 💰 **매수 신호 (Buy Signal)**

```
조건 1: 트렌드 확인
├─ EMA(12) > EMA(26) > SMA(50) ✓

조건 2: 모멘텀
├─ RSI(14) < 40 (과매도 구간) ✓
├─ MACD > Signal Line ✓
├─ MACD가 상승 중 (히스토그램 증가) ✓

조건 3: 거래량 검증 ⭐ (필수)
├─ 현재 거래량 > 20일 평균 × 1.3 이상 ✓

조건 4: 가격 액션 (옵션)
├─ 캔들 몸체 > 양봉 ✓

🟢 결론: 모든 조건 만족 → BUY 신호
신뢰도: ⭐⭐⭐⭐⭐ (95% 이상)
```

---

## 💸 **매도 신호 (Sell Signal)**

```
조건 1: 기본 매도 신호
├─ RSI(14) > 60 (과매수 구간) ✓
├─ MACD < Signal Line ✓
├─ MACD가 하락 중 (히스토그램 감소) ✓

조건 2: 거래량 검증 ⭐ (필수)
├─ 현재 거래량 > 20일 평균 × 1.5 이상 ✓

조건 3: 손익 관리
├─ 목표가 도달: 수익 실현 (자동 매도)
├─ 손절가 도달: 손실 제한 (자동 매도)

🔴 결론: 조건 만족 → SELL 신호
신뢰도: ⭐⭐⭐⭐ (90% 이상)
```

## 📈 **최적 거래량 활용 전략**

### 거래량 증가 → 신호 강화
```
상황 1: 상승 추세 + 거래량 급증
→ 강한 매수 기회 (기관 진입 신호)
→ 목표가 높게 설정

상황 2: 하락 추세 + 거래량 급증
→ 패닉 셀링 (매도 강도 높음)
→ 곧 반등 가능 (매수 찬스)

상황 3: 횡보 + 거래량 감소
→ 거래 피하기 (불확실성 높음)
```

### 거래량 프로필 (Volume Profile) 활용
```
고거래량 구간 = 지지선/저항선
→ 가격이 고거래량 구간에서 반등하는 경향
→ 그 구간 근처에서 거래 진입/청산
```

## 🛡️ **리스크 관리 (매우 중요)**

### 포지션 크기 관리
```
1회 최대 투자: 계정 잔액의 2-3%
예: 1,000만원 계정 → 1회 200-300만원만 투자

손절 설정:
- 진입가 대비 2-3% 손실 시 자동 청산
- 예: 100만원에 진입 → 97-98만원에서 손절

익절 설정:
- 진입가 대비 5-8% 수익 시 자동 청산
- 예: 100만원에 진입 → 105-108만원에서 익절
```

### 1일 최대 손실 제한
```
하루 최대 손실액 = 계정 잔액 × 5-10%
초과 시 당일 거래 중단
```

```

✓ 1일 봉 기준: 주 1-2회 거래 (장기 보유)

권장: 4시간 봉 + 1시간 봉 조합

## 🎲 **실제 거래 예시**

### 시나리오 1: 강한 상승 신호 (BTC 예시)
```
시간: 2024-01-15 오후 3시
현재가: $42,500 (1시간 봉)

조건 확인:
✓ EMA(12)=$42,400 > EMA(26)=$42,200 > SMA(50)=$42,000 (상승 추세)
✓ RSI=35 (과매도)
✓ MACD > Signal Line (상승 전환)
✓ 거래량=$500M > 20일 평균$300M × 1.5 =$450M ✓✓

→ 🟢 매수 신호!

매수 실행:
- 진입가: $42,500
- 손절가: $41,625 (2.5% 손실)
- 익절가: $45,375 (6.75% 수익)
- 포지션: 1BTC

결과: 5시간 후 → $45,600 도달 → 익절 실행
수익: $3,100 (7.3%)
```

### 시나리오 2: 거래량 부족 (피해야 할 신호)
```
시간: 2024-01-15 새벽 2시
현재가: $42,300

조건 확인:
✓ EMA(12)=$42,200 > EMA(26)=$42,100 > SMA(50)=$41,900 (상승 추세)
✓ RSI=38 (과매도)
✓ MACD > Signal Line (상승 전환)
❌ 거래량=$150M < 20일 평균$300M × 1.5=$450M (거래량 부족!)

→ 🟡 신호 신뢰도 낮음 → 거래 피하기!
이유: 거래량 없이 올라온 신호는 가짜 신호 확률 높음
```

```

```
1. MACD 계산
2. 거래량 이동평균
3. 신호 결합 로직
4. 손절/익절 자동화

3. 실시간 신호 강도 표시
4. 성과 분석 리포트
```

## 💡 **핵심 팁**

```
1. 거래량 > 모든 것
   → 거래량 없으면 거래하지 말 것!

2. 손절은 프로의 필수 항목
   → 손절 없는 거래는 사기꾼의 거래

3. 욕심내지 말 것
   → 5% 수익이 50%보다 낫다 (기회 많음)

4. 거래 기록 필수
   → 매 거래마다 왜 샀는지 이유 기록
   → 손실 거래 분석 → 개선

5. 감정 거래 금지
   → 자동화된 봇으로 실행하기

6. 정기적 백테스트
   → 매달 1회 지난 거래 분석
   → 파라미터 조정
```
